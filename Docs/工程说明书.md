# STM32F103C8T6 温度检测报警工程说明书

## 1. 工程概述

本工程用于 STM32F103C8T6 最小系统板，实现双路温度检测、阈值报警、OLED 显示、SD 卡记录、串口调参、阈值掉电保存和云服务预留。

核心功能：

- DS18B20 数字温度采集
- PT100 电压采集，按 `温度 = 51.2 * 电压` 换算
- OLED 实时显示温度、阈值、报警状态、SD 状态、最高/最低温
- 蜂鸣器和 LED 声光报警
- 三个按键调节 DS/PT 各自上下阈值
- 报警可手动静音
- 连续 2 次超阈值才报警，降低误报警
- SD 卡 FatFs 记录 CSV 文件
- SD 写入失败计数显示
- 阈值保存到 STM32 内部 Flash，断电不丢失
- 串口命令读取状态、设置阈值、恢复默认、静音
- ESP-01S 云服务流程保留，默认关闭
- 云上传失败时预留 SD 离线缓存流程

| 按键 2 | K2 | PB4 | GPIO Input Pull-up | 当前阈值增大 |

| ST-Link | SWDIO | PA13 | SWDIO | 下载/调试 |
| ST-Link | SWCLK | PA14 | SWCLK | 下载/调试 |
| 板载 LED | LED | PC13 | GPIO Output | 心跳灯 |

注意：

- PB3/PB4 默认属于 JTAG 引脚，本工程已关闭 JTAG，只保留 SWD。因此 CubeMX/Keil 中 Debug 必须选 `Serial Wire`。
- 所有模块必须共地。
- PT100 模块输出必须确认不超过 3.3V，否则会损坏 STM32 ADC。
- ESP-01S 必须使用 3.3V 供电，电流能力要足够。

## 4. OLED 显示说明

正常运行时 OLED 8 行含义如下：

```text
DS:+025.3C
PT:+051.2C
DSL:010 DSH:060
PTL:000 PTH:100
DA:0 PA:0 AL:0
M:0 DSL SD:1 E:0
D:024/026
P:050/053
```

逐行解释：

| 行 | 示例 | 含义 |
|---|---|---|
| 1 | `DS:+025.3C` | DS18B20 当前温度；`DS:ERR` 表示读取失败 |
| 2 | `PT:+051.2C` | PT 电压换算温度 |
| 3 | `DSL:010 DSH:060` | DS18B20 低阈值/高阈值 |
| 4 | `PTL:000 PTH:100` | PT 低阈值/高阈值 |
| 5 | `DA:0 PA:0 AL:0` | DS 报警/PT 报警/总报警 |
| 6 | `M:0 DSL SD:1 E:0` | 静音状态/当前调节项/SD 状态/SD 错误次数 |
| 7 | `D:024/026` | DS18B20 开机后的最低/最高温 |
| 8 | `P:050/053` | PT 开机后的最低/最高温 |

状态含义：

- `DA:1`：DS18B20 连续 2 次超阈值
- `PA:1`：PT 连续 2 次超阈值
- `AL:1`：总报警成立
- `M:1`：报警已手动静音
- `SD:1`：SD 卡正常
- `SD:0`：SD 卡未挂载或写入失败
- `E:n`：SD 写入失败次数
- `WF:0`：云服务关闭或未连接；当前默认关闭

OLED 采用“内容变化才刷新”的方式，正常不会快速闪屏。

## 5. 按键使用说明

三个按键均为低电平按下。

| 操作 | 功能 |
|---|---|
| `KEY3` 短按 | 切换当前调节项：`DSL -> DSH -> PTL -> PTH` |
| `KEY1` 短按 | 当前阈值减小 1 C |
| `KEY2` 短按 | 当前阈值增大 1 C |
| `KEY1` 长按 | 超过 1 秒后连续减小，每 250 ms 一次 |
| `KEY2` 长按 | 超过 1 秒后连续增大，每 250 ms 一次 |
| `KEY1 + KEY2` 同时按 | 报警静音/恢复 |

阈值说明：

- `DSL`：DS18B20 低温报警阈值
- `DSH`：DS18B20 高温报警阈值
- `PTL`：PT 低温报警阈值
- `PTH`：PT 高温报警阈值

阈值改动后会自动保存到 STM32 内部 Flash，断电后仍然保留。

## 6. 串口命令说明

USART1 用于串口调试：

- PA9：STM32 TX，接 USB-TTL RX
- PA10：STM32 RX，接 USB-TTL TX
- 波特率：115200
- 数据格式：8N1

支持命令：

```text
GET
SET DSL 10
SET DSH 60
SET PTL 0
SET PTH 100
RESET
MUTE
```

命令说明：

| 命令 | 功能 |
|---|---|
| `GET` | 查看当前阈值、SD 状态、报警状态 |
| `SET DSL 10` | 设置 DS18B20 低阈值为 10 C |
| `SET DSH 60` | 设置 DS18B20 高阈值为 60 C |
| `SET PTL 0` | 设置 PT 低阈值为 0 C |
| `SET PTH 100` | 设置 PT 高阈值为 100 C |
| `RESET` | 恢复默认阈值并保存 |
| `MUTE` | 报警静音/恢复 |

## 7. SD 卡记录说明

SD 卡使用 SPI2 + FatFs。

接线：

| SD 模块 | STM32 |
|---|---|
| CS | PB12 |
| SCK | PB13 |
| MISO/DO | PB14 |
| MOSI/DI | PB15 |
| VCC | 3.3V |
| GND | GND |

使用要求：

- SD 卡建议格式化为 FAT32。
- 上电前插入 SD 卡。
- 若 OLED 显示 `SD:0`，检查接线、电源、格式化和卡座接触。

日志文件：

```text
temp_log.csv
```

首次创建文件时会自动写表头：

```text
time_ms,ds18b20,pt100,ds_low,ds_high,pt_low,pt_high,ds_alarm,pt_alarm,alarm
```

记录周期：15 秒。

云服务离线缓存文件：

```text
cloud_cache.csv
```

当前云服务默认关闭，所以正常不会生成该文件。以后开启云服务后，如果上传失败，会把待上传数据缓存到该文件。

## 8. 报警逻辑

采样周期：15 秒。

判断逻辑：

1. 每 15 秒读取 DS18B20 和 PT。
2. 判断是否低于低阈值或高于高阈值。
3. 同一路温度连续 2 次超阈值，才认为报警成立。
4. 任一路报警成立，总报警 `AL` 置 1。
5. 总报警成立且未静音时，蜂鸣器和报警灯按节奏闪烁/鸣响。
6. 温度恢复正常后，报警和静音状态自动清除。

## 9. 云服务说明

ESP-01S 云服务代码已经保留，但默认关闭：

```c
#define APP_ENABLE_WIFI 0
```

配置文件：

```text
Inc/app_config.h
```

如需开启，需要填写：

```c
WIFI_SSID
WIFI_PASSWORD
ONENET_PRODUCT_ID
ONENET_DEVICE_NAME
ONENET_DEVICE_TOKEN
```

然后把：

```c
#define APP_ENABLE_WIFI 0
```

改成：

```c
#define APP_ENABLE_WIFI 1
```

云服务流程：

1. `ESP8266_Init()` 初始化 ESP-01S。
2. 连接 WiFi。
3. 连接 OneNET MQTT。
4. 每 15 秒调用 `ESP8266_Upload()` 上传温度和报警状态。
5. 如果上传失败，写入 SD 卡 `cloud_cache.csv`。
6. 网络恢复并上传成功后，清除缓存文件。

注意：ESP-01S 必须使用稳定 3.3V 电源，不建议直接从普通 USB-TTL 小电流 3.3V 脚供电。

## 15. 常见问题

### OLED 不显示

- 检查 OLED 地址是否为 0x3C。
- 检查 PB6/PB7 是否接反。
- 检查 OLED 电源和 GND。

### DS 显示 ERR

- 检查 PA0 接线。
- 检查 DS18B20 是否有 4.7k 上拉到 3.3V。
- 检查 DS18B20 电源方向。

### PT 温度不对

- 用万用表测 PA1 输入电压。
- 当前公式是 `T = 51.2 * V`。
- 例如 PA1 是 1.00V，则显示约 51.2 C。
- PA1 电压不能超过 3.3V。

### SD:0 或 E 变大

- SD 卡格式化为 FAT32。
- 检查 PB12/PB13/PB14/PB15。
- 检查 SD 模块是否支持 3.3V 逻辑。
- 重新插卡后按复位。

### 蜂鸣器一直响

- 查看 OLED 的 `DA/PA/AL`。
- 若 `AL:1`，说明连续 2 次超阈值。
- 可按 `KEY1 + KEY2` 静音。
- 检查阈值是否设置过低/过高。

### 按键没反应

- 按键一端接 GPIO，一端接 GND。
- 工程使用内部上拉，所以按下应为低电平。
- PB3/PB4 需要关闭 JTAG，只保留 SWD。
